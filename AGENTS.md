# AGENTS.md

本文件是本项目的长期执行约束。所有 Agent 在修改代码、文案、页面结构或视觉设计前，必须先阅读本文件，并优先遵守这里的项目定位、开发顺序和代码规范。

## 项目定位

项目名称：AI 二次元 / OC 创作平台网站。

一句话定位：面向二次元用户、OC 创作者、同人作者、Coser、摊主和小型创作者团队的 AI 创作工具平台，帮助用户从角色灵感快速生成 OC 设定、角色卡、头像方向、同人片段、摊宣文案、商品菜单和周边预览方案。

核心定位必须是：**AI 创作工具平台**。

当前阶段的主要目标是完成网站前端开发与视觉设计，先跑通可点击、可浏览、可演示的页面体验。后端、真实登录、数据库、支付、真实 AI 接口和供应链能力暂不作为当前阶段目标。

## 产品关键词

- AI 二次元创作
- OC 设定生成
- 角色卡生成
- 同人剧情辅助
- 角色对话风格
- AI 头像方向
- 周边定制预览
- 漫展摊宣工具
- 创作者资产管理
- 轻量角色互动

## 核心用户

1. 普通二次元用户：快速生成自己的 OC、人设、头像方向和角色卡，用于社交平台展示或自娱创作。
2. OC / 同人创作者：长期保存角色设定、世界观、关系网、剧情片段和角色说话风格。
3. Coser / 摊主 / 社团：生成漫展摊宣文案、商品菜单、价格牌、预售文案和社交平台宣传内容。
4. 小型创作者团队：用 AI 降低创作、宣发、设计和周边转化成本。

## 当前开发目标

现阶段优先完成一个高完成度的前端原型，而不是完整后端产品。

成功标准：

- 页面结构完整，用户能从首页进入主要功能页。
- 所有核心页面都有真实可感知的布局、内容层级和交互状态。
- 表单、按钮、筛选、Tab、卡片、详情页等前端交互可以演示。
- 生成、保存、删除、复制等行为可以先用前端 mock 数据或本地状态模拟。
- 页面在桌面端和移动端都可用。
- 视觉风格符合二次元创作工具定位，不像通用 SaaS 模板，也不偏陪聊网站。

## VGen 设计参考

本项目的前端设计参考 `/Users/luoshenghui/Downloads/ppqq/vgen-design-code-analysis.md` 中对 VGen 的分析。只学习其信息架构、组件组织和交互节奏，不复制 VGen 的 logo、品牌文案、图片素材、交易机制或具体业务。

可借鉴的方向：

- 首页即产品，不做冗长介绍型 landing page。
- 首屏要能直接进入创作、搜索、浏览示例内容。
- 用浅灰白背景承载高饱和内容资产，让角色卡、插画占位、周边预览和摊宣卡片成为视觉主角。
- 使用大圆角卡片、pill 搜索框、pill 按钮、圆形分类入口和横向滚动内容流。
- 桌面端保留完整顶部导航；移动端顶部简化，主导航可采用底部 tab bar。
- 首页多个 section 采用 discovery rail：横向浏览角色示例、创作模板、摊宣模板、周边方向。
- 分类和筛选使用 chips / tabs / orb row，先做前端本地筛选。
- 卡片需要包含可扫读信息：标题、标签、状态、简短说明、示例结果、收藏或保存入口。

不要照搬的内容：

- 不做 marketplace 交易首页。
- 不做真实订单、购物车、创作者后台、支付、多货币、私信或通知系统。
- 不复制 VGen 的品牌立场、文案、图片、图标和具体色值。
- 不把首页做成纯商品列表；本项目必须围绕“OC 创作、角色资产、摊宣工具”组织内容。

本项目首页应更接近“二次元创作发现页”：用户一进入页面就能看到创作入口、角色卡示例、模板分类和可继续探索的内容流。

## 前端 MVP 主路径

首版前端优先跑通完整演示链路：

```text
用户进入首页
→ 点击进入创作页
→ 输入角色灵感
→ 看到模拟生成的 OC 人设
→ 查看角色卡结果
→ 保存到前端角色库
→ 进入角色详情
→ 继续生成剧情 / 对话风格 / 头像提示词 / 摊宣文案 / 商品菜单
```

这一阶段可以使用 mock 数据、静态示例、前端状态或 localStorage。不要为了前端原型提前引入数据库、真实登录、支付或复杂 API。

## 代码规范

### 基本原则

- 先完成可用页面，再做抽象。
- 只写当前页面和交互需要的代码，不做超前封装。
- 组件拆分以复用和可读性为准，不为单次使用创建复杂抽象。
- 页面文件负责布局和组合，复杂 UI 片段下沉到组件。
- mock 数据集中管理，不要在多个组件里散落硬编码。
- 类型定义与实际 UI 使用保持一致，不提前设计后端字段。
- 不引入未说明的大型依赖。
- 不做无关格式化、无关重构或无关依赖升级。

### React / Next.js

- 优先使用现有项目技术栈；如果项目尚未开始，默认使用 Next.js + TypeScript + Tailwind CSS。
- 能用 Server Component 展示的静态内容不要强行改成 Client Component。
- 需要状态、事件、表单、localStorage 或浏览器 API 时再使用 Client Component。
- 组件 props 保持明确、简短，避免传入大而全对象后在组件内部猜字段。
- 列表渲染必须使用稳定 key。
- 表单必须有清晰的初始值、提交状态和错误提示。
- loading、error、empty、success 状态必须可见。

### CSS / Tailwind

- 优先使用项目已有设计系统、组件库和 Tailwind 配置。
- 不写大段重复 class；重复明显时再抽组件或提取样式。
- 保持移动端优先，所有核心页面必须适配小屏。
- 避免文字溢出、按钮挤压、卡片高度跳动和内容遮挡。
- 交互控件要有 hover、focus、disabled 状态。
- 视觉层级优先通过间距、字号、字重、色彩对比实现，不依赖过多装饰。

## 视觉与交互方向

整体风格参考 VGen 的轻量 marketplace / discovery 体验，但必须转译为二次元创作工具，而不是陪聊网站或交易平台。

推荐：

- 浅灰白页面背景，给高饱和角色卡和作品占位留出舞台
- 明亮、干净、有创作感
- 轻微 ACG 视觉元素，避免低质装饰
- 大圆角卡片、pill 搜索框、pill 按钮和圆形分类入口
- 横向滚动 rail 作为首页和移动端的核心浏览模型
- 高对比 CTA
- 角色卡预览突出，创作结果比平台装饰更抢眼
- 移动端优先
- 内容密度适中
- UI 克制，让角色卡、插画占位、模板卡片和创作结果成为视觉主角

建议设计 token：

```css
:root {
  --color-bg: #f8f7f8;
  --color-text: #0c090d;
  --color-muted: rgba(12, 9, 13, 0.62);
  --color-border: #eceaec;
  --color-card: #ffffff;
  --color-primary: #54c5b7;
  --color-pink: #f084bc;
  --color-blue: #6283d6;
  --color-purple: #946cf3;
  --color-lime: #b8ff26;
  --radius-pill: 999px;
  --radius-card: 20px;
  --radius-small: 8px;
}
```

可以参考这组 token 的方向，但不要求逐字使用。若项目已有主题变量，优先映射到现有主题。

避免：

- 过度粉色恋爱风
- 低俗陪聊广告风
- 过多闪烁动效
- 信息堆叠混乱
- 只展示聊天框，不展示创作工作流
- 复制 VGen 的品牌文案、图片素材或完整 marketplace 交易结构

交互要求：

- 用户点击生成后显示明确 loading 状态。
- 生成失败时展示可理解的错误信息。
- 结果为空时给出重新输入建议。
- 结果区域必须支持一键复制。
- 当前阶段保存结果可以使用前端状态或 localStorage。
- 角色卡应适合截图、分享和导出。
- 首页搜索框、分类 chips、tabs、横向 rail、收藏 / 保存按钮必须有可演示的前端状态。
- 移动端横向 rail 的卡片宽度应露出下一张，给用户明确滑动暗示。

角色卡至少展示：

- 角色名
- 头像占位区
- 标签
- 一句话设定
- 外貌
- 性格
- 背景
- 口癖
- 创作备注

摊宣输出适配：

- 小红书：标题、正文、标签，emoji 适度。
- 微博：短标题、正文、话题标签。
- QQ 群：直接、清晰，包含时间、地点和价格。
- B 站动态：适合创作者语气和展示说明。
